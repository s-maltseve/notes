# Git

Git - это система контроля версий. Она помогает сохранять историю изменений в проекте и загружать файлы на GitHub.

## Основные понятия

**Репозиторий** - папка проекта, за которой следит Git.

**Commit** - сохраненная версия изменений. Это как контрольная точка.

**GitHub** - сайт, где можно хранить репозитории онлайн.

**Push** - отправить изменения с компьютера на GitHub.

**Pull** - скачать последние изменения с GitHub на компьютер.

## Как обычно работать

Перед началом работы подтянуть свежие изменения:

```bash
git pull
```
## Основные команды Git

### `git status`

Показывает текущее состояние репозитория:

- какие файлы изменены
- какие файлы новые
- какие файлы уже подготовлены к commit
- есть ли несохраненные изменения

```bash
git status
```

### `git add`

Добавляет изменения в подготовку к commit.

Добавить все изменения:

```bash
git add .
```

Добавить конкретный файл:

```bash
git add git.md
```

### `git commit`

Сохраняет подготовленные изменения в истории Git.

Commit - это контрольная точка изменений.

```bash
git commit -m "обновил заметки"
```

### `git push`

Отправляет commits на GitHub.

После этой команды изменения появляются в репозитории на сайте.

```bash
git push
```

### `git pull`

Забирает последние изменения с GitHub на компьютер.

Эту команду полезно делать перед началом работы.

```bash
git pull
```

### `git clone`

Скачивает репозиторий с GitHub на компьютер.

```bash
git clone git@github.com:s-maltseve/notes.git
```

### `git branch`

Показывает список веток.

```bash
git branch
```

Создать новую ветку:

```bash
git branch feature/update-notes
```

### `git switch`

Переключает между ветками.

```bash
git switch main
```

Создать новую ветку и сразу перейти в нее:

```bash
git switch -c feature/update-notes
```

### `git merge`

Объединяет изменения из одной ветки в другую.

```bash
git switch main
git merge feature/update-notes
```

### `git diff`

Показывает, что именно изменилось в файлах.

```bash
git diff
```

Посмотреть изменения в конкретном файле:

```bash
git diff git.md
```

### `git log`

Показывает историю commit.

```bash
git log
```

Краткий вид:

```bash
git log --oneline
```

Чтобы выйти из просмотра истории, нажать:

```text
q
```

### `git remote -v`

Показывает, к какому GitHub-репозиторию подключена текущая папка.

```bash
git remote -v
```

## Типичный порядок работы с заметками

Когда начинаешь работать:

```bash
git pull
```

Когда закончил редактировать заметки:

```bash
git status
git add .
git commit -m "обновил заметки"
git push
```

## Важно помнить

Если файл изменен только на компьютере, но не сделан `git push`, то на GitHub этих изменений еще нет.

Если работаешь с заметками с разных компьютеров, перед началом всегда делай `git pull`.

---

Зачем DevOps'у Git и почему он в начале?

Все конфиги, пайплайны, манифесты, YAML-файлы — это код. Этот код должен где-то храниться, версионироваться и меняться.

При этом неважно, GitHub, GitLab или BitBucket, потому что все они строятся на git.
